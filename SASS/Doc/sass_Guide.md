# Sass Guideline


### 왜 스타일가이드가 필요한가?

큰 프로젝트 일수록 코딩 스타일가이드는 필수. 하지만 작은 프로젝트일지라도 코드 품질 관리에 있어 가이드라인 역할은 비중이 크다.


### 핵심 원칙
스타일 가이드에서 요구하는 포인트! __Sass사용 방법이 가능한 간단하게 유지되어야 함__
css 자체가 간단 명료. 이것이 복잡해지면 안됨.
> css처럼 Sass 작성 또한 쉽고 간결해야함, 

### 구문/서식

- space 2칸 들여쓰기
- 이상적인 행 너비는 80글자
- 적절하게 쓰인 여러 행의 CSS 규칙
- 공백의 의미있는 사용 (가독성)

```sass
.app
  display: block
  overflow: hidden
  padding: 0 1em
```
